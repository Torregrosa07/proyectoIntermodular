¿Qué es el Software?

Para entender qué es el software, imaginemos un ordenador. El software es como la mente que le da vida al cuerpo (hardware). La RAE lo describe simplemente como un conjunto de programas e instrucciones que la computadora usa para hacer tareas. Pressman, en cambio, lo define con más detalle, incluyendo las instrucciones (programas), la forma en que se organizan los datos y la documentación que explica cómo funciona.

¿Cómo funciona?

El ordenador tiene tres partes principales:

  1. Unidad Central de Proceso (CPU)

    Ejecuta las instrucciones de los programas, traduciendo operaciones complejas en operaciones aritméticas y lógicas básicas.

    • Unidad Aritmético-Lógica (ALU): Ejecuta operaciones aritméticas y lógicas
    • Unidad de Control (UC): Recoge y ordena la ejecución de instrucciones
    • Registros: Almacenamiento interno de la CPU

  2. Memoria Principal (RAM)

    Contiene las instrucciones del programa y los datos. Es una memoria volátil que se borra al apagar el ordenador.

  3. Unidad de Entrada/Salida

    Permite la comunicación del ordenador con el exterior mediante diversos periféricos:

    • Periféricos de Entrada: Teclado, ratón
    • Periféricos de Salida: Pantalla, impresora
    • Periféricos de Entrada/Salida: Módem, tarjeta de red, dispositivos de almacenamiento

El software son las instrucciones que se guardan en la memoria RAM. La CPU las lee y las ejecuta, usando los datos que también están en la memoria. Los resultados se pueden mostrar en la pantalla o guardar en el disco duro.

Tipos de software

  • Software de sistemas: Es el sistema operativo, como Windows o macOS, que permite usar el ordenador e interactuar con él.
  • Software de programación: Son las herramientas que usan los programadores para crear nuevo software.
  • Software de aplicación: Son los programas que usamos a diario, como procesadores de texto, navegadores web o videojuegos.

En resumen, el software es lo que hace que un ordenador sea útil. Son las instrucciones que le dicen qué hacer, cómo procesar la información y cómo interactuar con nosotros.

Conceptos de Código

Código Fuente

El código fuente es el conjunto de instrucciones que los programadores escriben en un lenguaje de programación para hacer que un programa o aplicación funcione. Es como el "guion" que le dice a la computadora qué hacer. Este código se puede modificar y mejorar para que el programa haga lo que necesitamos.

Código Objeto

Es el resultado de convertir el código fuente, que es escrito por los programadores, a un formato que la computadora puede entender, pero aún no es completamente ejecutable. Es un archivo intermedio que contiene instrucciones en un lenguaje de bajo nivel, generalmente en código binario, pero necesita pasar por un proceso de enlace (linking) para unirse con otros archivos de código objeto y formar el programa final. Una vez que el código objeto se enlaza correctamente, se convierte en un archivo ejecutable que la computadora puede correr sin problemas.

Código Ejecutable

Es el archivo final que la computadora puede ejecutar directamente. Es el resultado de compilar y vincular todo el código fuente y el código objeto. Este tipo de archivo ya está en un formato que el sistema operativo y la máquina pueden entender y ejecutar sin necesidad de ningún programa intermedio. Los archivos ejecutables suelen tener extensiones como .exe en Windows o .out en sistemas Unix/Linux. Cuando abres un archivo ejecutable, el programa comienza a funcionar, siguiendo las instrucciones que se definieron en el código fuente original.

Tipos de Traductores

Traductores:

Para realizar esta traducción del código fuente al código objeto, se utilizan dos tipos de traductores:

1. Compiladores:

  • Traducen todo el código fuente de una vez.
  • Analizan el código en busca de errores.
  • Si no hay errores, generan un archivo de código objeto.
  • Dependiendo del compilador, el código objeto puede ser ejecutable o requerir pasos adicionales (ensamblado, enlazado).
  • Una vez que se obtiene el código ejecutable, se puede ejecutar sin necesidad de volver a compilar.

2. Intérpretes:

  • Traducen y ejecutan el código línea por línea.
  • No generan un archivo de código objeto permanente.
  • Cada fragmento de código fuente se traduce y ejecuta en el momento.

El caso de Java:

Java combina compilación e interpretación. Primero, el código fuente se compila a un formato intermedio llamado "bytecode". Luego, este bytecode es interpretado por la máquina virtual de Java (JVM).

Herramientas:

Para programar, se necesitan herramientas como editores de texto, compiladores o intérpretes, y enlazadores. Estas herramientas suelen estar integradas en entornos de desarrollo (IDEs), que facilitan la escritura, traducción y ejecución del código. Los IDEs también incluyen depuradores, herramientas de control de versiones y constructores de interfaces gráficas.

Proceso de Obtención de Código Ejecutable a partir de Código Fuente

El proceso de obtención de código ejecutable a partir de código fuente es una secuencia de pasos crucial para transformar un programa escrito en un lenguaje de programación de alto nivel (como C, Java, Python) en un archivo ejecutable que la computadora pueda ejecutar directamente. Este proceso implica varias etapas y el uso de herramientas específicas que convierten el código legible por los humanos en un formato que el hardware de la computadora puede interpretar.

Entre las herramientas implicadas en este proceso se encuentran:

    Escritura del Código Fuente: El programador escribe el código utilizando un editor de texto o un IDE.

    Compilación:El compilador convierte el código fuente en código objeto, verificando errores de sintaxis y transformando las instrucciones en un formato de bajo nivel.

    Enlazado (Linking): El enlazador combina el código objeto con las bibliotecas necesarias para generar el archivo ejecutable final.

    Generación del Archivo Ejecutable: El resultado es un archivo que la computadora puede ejecutar, como un archivo .exe o .out.

    Depuración y Optimización: El uso de herramientas como depuradores (e.g., GDB) y optimizadores mejora la eficiencia y corrige posibles errores en el programa.

El proceso de obtención de un código ejecutable implica desde la redacción del código hasta la generación de un archivo que la computadora puede ejecutar, utilizando herramientas como compiladores, intérpretes y enlazadores.

Lenguajes de Programación

Los ordenadores solo entienden el lenguaje binario (ceros y unos), pero programar directamente en este lenguaje es muy difícil. Por eso, se crearon los lenguajes de programación, que permiten a los programadores escribir instrucciones en un lenguaje más cercano al humano.

Clasificación de los lenguajes de programación

Los lenguajes de programación se pueden clasificar según su cercanía al lenguaje máquina:

  • Lenguajes de bajo nivel (lenguaje máquina): Son los más cercanos al lenguaje binario y los más difíciles de usar para los humanos. Fueron los primeros en utilizarse, pero se abandonaron por su complejidad.
  • Lenguajes intermedios (lenguajes ensambladores): Utilizan códigos simbólicos en lugar de números binarios, lo que los hace más fáciles de entender. Cada instrucción en ensamblador se traduce a una instrucción en lenguaje máquina.
  • Lenguajes de alto nivel: Son los más utilizados actualmente. Permiten a los programadores escribir código de forma más sencilla y rápida, sin necesidad de conocer detalles del hardware. Ejemplos: Java, C++, Python.

Elementos básicos de un lenguaje de programación

  • Identificadores: Nombres que se dan a los elementos del programa (variables, funciones, etc.).
  • Constantes: Datos que no cambian durante la ejecución del programa.
  • Operadores: Símbolos que realizan operaciones (suma, resta, etc.).
  • Instrucciones: Órdenes que se dan al ordenador.
  • Comentarios: Texto que explica el código.

Paradigmas de programación

Dentro de los lenguajes de alto nivel, existen dos paradigmas principales:

  • Lenguajes estructurados: Se basan en tres estructuras básicas (secuencial, alternativa y repetitiva). El programa se divide en módulos que se llaman entre sí. Ejemplos: Pascal, C, COBOL.
  • Lenguajes orientados a objetos: Se basan en la interacción entre objetos. Un objeto tiene datos (atributos) y acciones (métodos). Los objetos se comunican mediante mensajes. Ejemplos: Java, C++, C#.

Ventajas de los lenguajes de alto nivel

  • Más fáciles de aprender y usar.
  • Permiten crear programas más complejos.
  • Mayor productividad.

Desventajas de los lenguajes de alto nivel

  • Necesitan ser traducidos a lenguaje máquina.
  • Mayor consumo de recursos.
  • Menor eficiencia que los lenguajes de bajo nivel.

Características principales:

  • Precisos: Cada instrucción tiene un significado único y específico para la computadora.
  • Sintaxis: Como las reglas gramaticales en español, los lenguajes de programación tienen reglas sobre cómo se deben escribir las instrucciones.
  • Diversos: Hay muchos lenguajes de programación diferentes, cada uno con sus propias fortalezas y debilidades. Algunos ejemplos populares son Python, Java, C++ y JavaScript.

¿Para qué se usan?

Los lenguajes de programación se utilizan para crear todo tipo de cosas, como:

  • Aplicaciones móviles: Las apps que usas en tu teléfono.
  • Sitios web: Las páginas que visitas en internet.
  • Videojuegos: ¡Desde juegos simples hasta mundos virtuales complejos!
  • Programas de computadora: Como procesadores de texto, hojas de cálculo y navegadores web.

Máquinas Virtuales: Concepto y Efectividad

¿En qué consiste la máquina virtual?

Una máquina virtual (VM) es un software que simula un sistema de computación integral, permitiendo el funcionamiento de un sistema operativo y programas como si estuvieran integrados en una computadora física independiente. Es básicamente un ordenador que se inserta en otro.

1. Máquinas Virtuales de Sistema

Estas aplicaciones emulan completamente un ordenador, lo que permite instalar en su interior otro sistema operativo con sus propios componentes, como disco duro, memoria, tarjeta gráfica, etc. La experiencia al trabajar con una máquina virtual de sistema es similar a la de una máquina real. Ejemplos destacados son VMware Workstation y VirtualBox.

Estas máquinas son útiles para disponer de varios sistemas operativos en un mismo equipo, instalando uno en el hardware físico y otro en la máquina virtual. Este enfoque resulta ideal para evaluar nuevos sistemas o probar aplicaciones en distintas plataformas. Sin embargo, es importante señalar que estas máquinas requieren recursos significativos del ordenador físico, especialmente en términos de memoria RAM y espacio en disco.

2. Máquinas Virtuales de Proceso

Este tipo de máquina virtual está diseñado para ejecutar un proceso específico dentro de un sistema operativo. La máquina se activa al iniciar el proceso y se detiene al concluir. Su propósito principal es proporcionar un entorno de ejecución que sea independiente del hardware y del sistema operativo, haciendo que los programas funcionen de manera uniforme en cualquier plataforma.

Un ejemplo destacado de este tipo de máquinas es la Máquina Virtual de Java (JVM, Java Virtual Machine), que hace posible que los programas escritos en Java sean portátiles. Esto se logra porque los programas no son ejecutados directamente por el procesador, sino por la JVM.

Razón de ser de la ingeniería del software

La ingeniería de software surgió como respuesta a la creciente complejidad de los sistemas informáticos y a la necesidad de su desarrollo eficiente y confiable. Su objetivo principal es proporcionar una forma sistemática y consistente de diseñar, construir y mantener un mejor software.

Objetivos básicos:

  1. Mejora la calidad del software: asegura de que el software sea seguro, eficiente y satisfaga las necesidades de los usuarios.
  2. Mejorar el proceso de desarrollo: Utiliza métodos y procedimientos que aumenten la eficiencia y reduzcan los costos y el tiempo de desarrollo.
  3. Gestión de la complejidad: proporciona recursos y métodos para gestionar grandes proyectos de software.
  4. Incrementar la mantenibilidad: Desarrollar software que se actualice y adapte fácilmente a las nuevas necesidades.
  5. Fomenta la reutilización: cree componentes de software que puedan usarse en múltiples proyectos.

Relevancia en el Contexto Actual

En un tiempo en el que el software se encuentra en todas partes, desde aplicaciones móviles hasta sistemas cruciales de control, la ingeniería del software es fundamental para:

  • Asegurar la seguridad y la confianza en sistemas esenciales.
  • Fomentar la innovación tecnológica.
  • Optimizar la experiencia del usuario en aplicaciones diarias.
  • Disminuir gastos y peligros en proyectos de creación de software.

Fases del desarrollo de una aplicación informática

Son las distintas etapas del proceso mediante las cuales se planifica, diseña, desarrolla, prueba, implementa y mantiene un software. Estas fases aseguran que el desarrollo se realice de manera estructurada y controlada, permitiendo la entrega de un producto de calidad que cumpla con las expectativas y necesidades del cliente.

El desarrollo de una aplicación informática implica una serie de fases para asegurar que el software resultante sea de calidad y cumpla con las necesidades del cliente. Estas fases son:

  1. Análisis:

    El objetivo de esta tarea es analizar las necesidades de los usuarios potenciales del software para determinar qué debe hacer la aplicación y, de acuerdo con ello, escribir una especificación precisa de dicho sistema. Durante esta fase se pretende responder a las siguientes preguntas: qué información ha de ser procesada, qué función y rendimiento se desean, qué interfaces deben establecerse, qué restricciones de diseño existen y qué criterios de validación se necesitan para definir un sistema correcto. Para ello, como es obvio, es necesario realizar una actividad inicial de comunicación con el cliente que ha encargado el desarrollo de la aplicación, y una vez obtenida la información necesaria, se podrá modelar el sistema. Como resultado, se obtendrá una especificación del sistema, que consistirá en una documentación que describe lo que tiene que hacer el software, pero no cómo. Es primordial que esta documentación sea comprensible, completa y fácil de verificar y de modificar para facilitar todo el trabajo posterior.

  2. Diseño:

    Si bien el objetivo de la fase de análisis es determinar qué debe hacer el sistema, una vez se sabe esto, es preciso establecer cómo se va a resolver el problema planteado. Durante esta fase, se traducen los requisitos resultado de la fase de análisis en componentes de software (tablas de una base de datos, programas con procedimientos o funciones, clases con atributos y métodos, interfaz de usuario, etc.). Se trata de un refinamiento de la fase anterior. Por ejemplo, en esta fase, una de las decisiones que habrá que tomar es decidir qué sistema gestor de base de datos (SGBD) se va a emplear para gestionar la información que debe ser usada o generada por la aplicación: Oracle, MySQL, SQL Server, etcétera.

  3. Programación:

    Consiste en traducir los resultados de la fase de diseño en una forma legible para la máquina, es decir, se escribe el código fuente de cada componente del software empleando un determinado lenguaje de programación. El resultado de esta etapa es el código ejecutable.

  4. Pruebas:

    con estas se comprueba si la aplicación funciona correctamente. Hay diferentes tipos de pruebas. Así, se comienza probando cada componente de software por separado (pruebas unitarias) y, posteriormente, se van integrando poco a poco los componentes (pruebas de integración), hasta probar el programa completo (pruebas de validación), cuyo objetivo es comprobar que la aplicación satisface los requisitos del cliente. Como resultado de estas pruebas, se descubrirán errores, en atención a los cuales, será necesario modificar el código, o incluso rehacer las tareas previas de diseño y/o análisis.

  5. Explotación:

    En esta etapa, se lleva a cabo la instalación y puesta en marcha del software en el entorno de trabajo del cliente.

  6. Mantenimiento:

    El software sufrirá cambios después de que se entregue al cliente por diversos motivos. La tarea de mantenimiento consiste en realizar cambios sobre el software, para lo que se aplicará al programa existente, cada una de las actividades precedentes del ciclo de vida en vez de a uno nuevo.

¿Qué Roles o Perfiles Intervienen?

Puesto que el desarrollo de software se puede dividir en distintas tareas, es habitual contar con una persona experta para cada una de ellas, lo que origina que en el campo de la ingeniería del software existan diferentes roles:

  • Jefe de proyecto:

    Lleva a cabo la tarea de planificación del proyecto, que incluye, además de la propia planificación, la puesta en marcha del proyecto, su ejecución, seguimiento, control y cierre. Se trata, pues, de una tarea más de gestión que técnica pero de gran relevancia, pues puede influir de manera decisiva en el éxito o fracaso del proyecto. Se trata del máximo responsable de que el proyecto salga adelante.

  • Expertos del dominio:

    Son personas que trabajan en la empresa u organismo para el que hay que desarrollar la aplicación y que conocen el dominio del problema, es decir, la parte de la realidad para la que se va a crear la aplicación. Los técnicos deben comunicarse con estas personas, puesto que son las que conocen en detalle los requisitos que la aplicación debe satisfacer.

  • Analista:

    Su cometido es la tarea de análisis, que consiste en crear un modelo claro y consistente que se corresponda con la visión del problema que ha conseguido con la ayuda de los expertos del dominio. Este rol también se conoce frecuentemente como analista funcional.

  • Arquitecto:

    A partir del trabajo realizado por el analista, debe definir las líneas maestras del diseño, estableciendo la arquitectura del sistema, esto es, la manera en que este se divide en una serie de componentes. Esta arquitectura deberá ser tomada como referencia por el resto de personas desarrolladoras para llevar a buen término su trabajo.

  • Diseñador:

    Parte de un subconjunto de los requisitos definidos por el analista y de la arquitectura del sistema para diseñar las partes del sistema que implementan esos requisitos hasta un nivel de detalle suficiente para acometer la tarea de programación. Este rol también se conoce como analista orgánico o analista técnico.

  • Programador:

    Se encarga de la tarea de programación, es decir, de escribir el código fuente partiendo del diseño detallado realizado por el diseñador.

  • Probador:

    Se encarga de la tarea de pruebas y, por tanto, de garantizar la calidad de la aplicación creada. Para ello partirá de los requisitos y de los programas, y se encargará de probarlos para decidir, en último término, si están en condiciones de ser entregados al usuario final.

  • Encargado de la implantación:

    Es el encargado de que se realice el empaquetado de los programas y su instalación en el entorno de trabajo del cliente. También es el encargado de gestionar la configuración y, por tanto, las diferentes versiones que vayan surgiendo de la aplicación a partir de los cambios que se vayan incorporando.

Modelos de ciclo de vida del software

El desarrollo de software implica varias tareas, como análisis, diseño, programación y pruebas, pero la forma en que se organizan estas tareas puede variar según el proyecto. A esto se le llama ciclo de vida del software.

Un ciclo de vida define el orden en que se realizan las tareas y los criterios para pasar de una a otra. Es como una guía que indica qué hacer en cada etapa y cómo saber si se está listo para avanzar.

Elegir el ciclo de vida adecuado es crucial para el éxito del proyecto. El primer modelo fue el ciclo de vida clásico o en cascada, donde las tareas se realizan en secuencia. Luego surgieron otros modelos, como el incremental, el evolutivo y el ágil, que ofrecen mayor flexibilidad.

Modelo en Cascada

Este es el modelo más antiguo y tradicional.

  • Características: Secuencial y lineal
  • Fases: Requisitos, Diseño, Implementación, Verificación, Mantenimiento
  • Ventaja: Simple y fácil de entender
  • Desventaja: Poca flexibilidad para cambios

Modelo Incremental

Este modelo desarrolla el software en pequeños incrementos funcionales.

  • Características: Entregas parciales y funcionales en cada incremento
  • Fases: Requisitos iniciales, Desarrollo y entrega, Evaluación y retroalimentación
  • Ventaja: Mayor flexibilidad y entregas rápidas
  • Desventaja: Planificación compleja y propagación de errores si no se detectan a tiempo

Modelo en Espiral

Se enfoca en un desarrollo iterativo y la gestión de riesgos a través de ciclos repetitivos.

  • Características: Iterativo, con enfoque en la gestión de riesgos
  • Fases: Identificación de objetivos, Análisis de riesgos, Desarrollo y verificación, Planificación del próximo ciclo
  • Ventaja: Ideal para proyectos grandes y complejos
  • Desventaja: Costoso y requiere experiencia en gestión de riesgos

Metodologías de desarrollo de software actuales

El desarrollo de software moderno se basa en diversas metodologías que buscan optimizar el proceso y la calidad del producto. Algunas de las más relevantes son:

Según Maddison (1983), una metodología es un conjunto de enfoques, fases, procedimientos, reglas, técnicas, herramientas y documentación, además de aspectos formativos destinados a quienes desarrollan sistemas de información. Mientras que el ciclo de vida define qué se busca obtener en cada etapa del proyecto, una metodología detalla cómo se realizan las tareas, qué herramientas usar, y cómo se organizan las actividades.

Conceptos clave

  • Proceso de desarrollo: Se organiza en fases donde se realizan diversas tareas.
  • Tareas: Cada tarea sigue un procedimiento que genera productos intermedios (útiles para tareas futuras) o finales (entregados al cliente).
  • Métodos y herramientas: Los métodos estructuran procedimientos mediante diagramas y textos. Las herramientas automatizan parcialmente estas tareas; un ejemplo son las herramientas CASE (Computer-Aided Software Engineering).

Evolución de las metodologías

  1. Desarrollo artesanal: En sus inicios, el desarrollo de software era no estructurado, lo que condujo a la "crisis del software".
  2. Metodologías estructuradas: Surgieron para regular la programación mediante enfoques estructurados para diseño y análisis.
  3. Metodologías orientadas a objetos: En los años 80 se introdujeron lenguajes orientados a objetos y métodos basados en este paradigma, centrados en los datos y su comportamiento en lugar de los procesos.

Proceso Unificado de Rational (RUP)

El proceso RUP combina dos dimensiones:

  • Temporal: Incluye ciclos, fases, iteraciones e hitos.
  • Estática: Relaciona actividades, productos intermedios, roles y flujos de trabajo.

Cada ciclo incluye cuatro fases:

  1. Inicio: Analiza la viabilidad del sistema, define objetivos, alcance, riesgos y recursos.
  2. Elaboración: Establece la base arquitectónica, planifica el proyecto y aborda los riesgos más importantes.
  3. Construcción: Desarrolla el sistema de manera iterativa e incremental.
  4. Transición: Finaliza el sistema, abordando ajustes y problemas emergentes para preparar su uso en producción.

Además, RUP incluye actividades como modelado del negocio, gestión de requisitos, diseño, implementación, pruebas, y despliegue, agrupadas en flujos de ingeniería y apoyo.

Metodologías ágiles

Las metodologías ágiles enfatizan la entrega continua de software funcional, priorizando la colaboración y la adaptabilidad sobre procesos rígidos. Entre sus principios se destacan:

  • Valorar a los individuos y las interacciones sobre los procesos y herramientas.
  • Producir software operativo más que documentación exhaustiva.
  • Colaborar con el cliente en lugar de negociar contratos.
  • Adaptarse al cambio en lugar de seguir estrictamente un plan.

Programación Extrema (XP)

Propuesta por Kent Beck, XP se basa en valores como simplicidad, comunicación, retroalimentación, valentía y respeto. Este enfoque fomenta:

  • Diseños simples y refactorización cuando sea necesario.
  • Comunicación continua y cercana con el cliente.
  • Retroalimentación frecuente mediante entregas incrementales.
  • Pruebas continuas y programación en pareja.

Scrum

Scrum, desarrollado por Jeff Sutherland, se enfoca en la entrega de valor al cliente mediante ciclos cortos llamados sprints. Sus pilares son:

  • Transparencia: Todos conocen el estado y progreso del proyecto.
  • Inspección: Autoevaluación constante por parte del equipo.
  • Adaptación: Cambios flexibles para cumplir los objetivos.

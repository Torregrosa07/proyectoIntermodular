Concepto de Software

Se conoce como software o soporte lógico al sistema formal de un sistema informático, que comprende el conjunto de los componentes lógicos necesarios que hace posible la realización de tareas específicas, en contraposición a los componentes físicos que son llamados hardware. La interacción entre el software y el hardware hace operativo un ordenador (u otro dispositivo), es decir, el software envía instrucciones que el hardware ejecuta, haciendo posible su funcionamiento.

Conceptos de Código

Código Fuente

El código fuente es el conjunto de instrucciones escritas en un lenguaje de programación que un programador crea para que una computadora pueda entender y ejecutar una tarea específica.

Código Objeto

El código objeto es el código que resulta de la compilación del código fuente. Durante la compilación, el compilador traduce el código fuente (escrito en un lenguaje de alto nivel) a un lenguaje de bajo nivel, generalmente lenguaje de máquina o lenguaje intermedio, que puede ser entendido por la máquina, pero no es directamente ejecutable aún.

Código Ejecutable

El código ejecutable es el archivo final que puede ser ejecutado directamente por la computadora. Es el código objeto que ha sido procesado por un enlazador (linker), que lo combina con bibliotecas necesarias y otros archivos de código objeto para generar un archivo independiente que puede correr en un sistema operativo determinado.

Lenguajes de Programación

Imagina que quieres decirle a una computadora qué hacer. No puedes simplemente hablarle en español o inglés, ¿verdad? Necesitas un lenguaje que la computadora entienda. ¡Ahí es donde entran los lenguajes de programación!

Básicamente, un lenguaje de programación es como un conjunto de instrucciones especiales que le damos a una computadora para que realice tareas específicas.

Características principales:

  • Precisos: Cada instrucción tiene un significado único y específico para la computadora.
  • Sintaxis: Como las reglas gramaticales en español, los lenguajes de programación tienen reglas sobre cómo se deben escribir las instrucciones.
  • Diversos: Hay muchos lenguajes de programación diferentes, cada uno con sus propias fortalezas y debilidades. Algunos ejemplos populares son Python, Java, C++ y JavaScript.

¿Para qué se usan?

Los lenguajes de programación se utilizan para crear todo tipo de cosas, como:

  • Aplicaciones móviles: Las apps que usas en tu teléfono.
  • Sitios web: Las páginas que visitas en internet.
  • Videojuegos: ¡Desde juegos simples hasta mundos virtuales complejos!
  • Programas de computadora: Como procesadores de texto, hojas de cálculo y navegadores web.

Proceso de Obtención de Código Ejecutable a partir de Código Fuente

El proceso de obtención de código ejecutable a partir de código fuente es el conjunto de pasos que se siguen para transformar un programa escrito en un lenguaje de programación de alto nivel (como C, Java o Python) en un archivo que la computadora pueda ejecutar directamente. Este proceso se lleva a cabo mediante el uso de diversas herramientas.

Entre las herramientas implicadas en este proceso se encuentran:

  • Editores de texto/IDEs (como Visual Studio Code, Eclipse, o PyCharm).
  • Compiladores (como GCC para C/C++).
  • Intérpretes (como Python o Node.js para JavaScript).
  • Enlazadores (linkers) y depuradores (como GDB).

El proceso de obtención de un código ejecutable implica desde la redacción del código hasta la generación de un archivo que la computadora puede ejecutar, utilizando herramientas como compiladores, intérpretes y enlazadores.

Máquinas Virtuales: Concepto y Efectividad

¿En qué consiste la máquina virtual?

Una máquina virtual (VM) es un software que simula un sistema de computación integral, permitiendo el funcionamiento de un sistema operativo y programas como si estuvieran integrados en una computadora física independiente. Es básicamente un ordenador que se inserta en otro.

Características Fundamentales

  • División: Opera de manera independiente del sistema operativo de destino.
  • Controlabilidad: Es posible mover o duplicar entre diferentes dispositivos.
  • Capacidad de Adaptación: Promueve la operación de múltiples sistemas operativos en un solo dispositivo de hardware.

Principales Usos

  • Desarrollo de Programas informáticos
  • Evaluar aplicaciones en diversos contextos
  • Establecer entornos de crecimiento seguros
  • Salud y Seguridad: Aislar aplicaciones que podrían ser potencialmente peligrosas. Ejecutar ensayos sin poner en riesgo el sistema principal.
  • Optimización de Capacidades de Recursos: Implementar múltiples servidores en un solo dispositivo físico. Disminuir los gastos de infraestructura.

Razón de ser de la ingeniería del software

La ingeniería de software surgió como respuesta a la creciente complejidad de los sistemas informáticos y a la necesidad de su desarrollo eficiente y confiable. Su objetivo principal es proporcionar una forma sistemática y consistente de diseñar, construir y mantener un mejor software.

Objetivos básicos:

  1. Mejora la calidad del software: asegura de que el software sea seguro, eficiente y satisfaga las necesidades de los usuarios.
  2. Mejorar el proceso de desarrollo: Utiliza métodos y procedimientos que aumenten la eficiencia y reduzcan los costos y el tiempo de desarrollo.
  3. Gestión de la complejidad: proporciona recursos y métodos para gestionar grandes proyectos de software.
  4. Incrementar la mantenibilidad: Desarrollar software que se actualice y adapte fácilmente a las nuevas necesidades.
  5. Fomenta la reutilización: cree componentes de software que puedan usarse en múltiples proyectos.

Relevancia en el Contexto Actual

En un tiempo en el que el software se encuentra en todas partes, desde aplicaciones móviles hasta sistemas cruciales de control, la ingeniería del software es fundamental para:

  • Asegurar la seguridad y la confianza en sistemas esenciales.
  • Fomentar la innovación tecnológica.
  • Optimizar la experiencia del usuario en aplicaciones diarias.
  • Disminuir gastos y peligros en proyectos de creación de software.

Fases del desarrollo de una aplicación informática

Análisis de Requisitos

Esta fase inicial es crucial para el éxito del proyecto. Su cometido es:

  • Identificar las necesidades del cliente y usuarios
  • Definir los objetivos y alcance del proyecto
  • Recopilar y documentar los requisitos funcionales y no funcionales
  • Establecer las restricciones y limitaciones del sistema

Diseño

Una vez definidos los requisitos, se procede al diseño. Esta fase tiene como cometido:

  • Crear la arquitectura del sistema
  • Diseñar la estructura de datos
  • Definir las interfaces de usuario
  • Establecer los componentes del software y sus relaciones

Modelos de ciclo de vida del software

Modelo en Cascada

Este es el modelo más antiguo y tradicional.

  • Características: Secuencial y lineal
  • Fases: Requisitos, Diseño, Implementación, Verificación, Mantenimiento
  • Ventaja: Simple y fácil de entender
  • Desventaja: Poca flexibilidad para cambios

Metodologías de desarrollo de software actuales

El desarrollo de software moderno se basa en diversas metodologías que buscan optimizar el proceso y la calidad del producto. Algunas de las más relevantes son:

1. Metodologías Ágiles

Se caracterizan por su flexibilidad y enfoque en la entrega continua de valor.

  • Scrum: Desarrollo iterativo en sprints cortos, con roles clave como Product Owner, Scrum Master y Equipo de Desarrollo.
  • Kanban: Visualización del flujo de trabajo, límite en el trabajo en progreso para optimizar el flujo.
  • Extreme Programming (XP): Prioriza prácticas de ingeniería como la programación en parejas y el desarrollo guiado por pruebas.

2. DevOps

Integra el desarrollo y las operaciones para automatizar procesos y acelerar la entrega de software.

3. Lean Software Development

Se centra en la eliminación de desperdicios y la optimización del flujo de valor, inspirada en los principios de manufactura Lean de Toyota.

4. Design Thinking

Metodología centrada en el usuario para diseñar soluciones innovadoras.

La elección de la metodología adecuada dependerá del proyecto, las necesidades del cliente y el equipo de desarrollo.
